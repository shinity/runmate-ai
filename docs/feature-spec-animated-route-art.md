# Feature Spec: Animated Route Art (애니메이션 라우트 아트)

## 1. 전략 요약

| 항목 | Route Art (기존) | Animated Route Art (신규) |
|------|-----------------|--------------------------|
| 결과물 형태 | 정적 SVG 이미지 | 캐릭터가 경로를 따라 달리는 애니메이션 GIF |
| 배경 | 단색/그라디언트 | AI 생성 이미지 (FLUX.1) 또는 프리셋 |
| 캐릭터 | 없음 | 프리셋 스프라이트 (픽셀아트 러너) |
| 공유 가치 | 경로 자체의 심미성 | 경로 + 캐릭터 + AI 배경의 복합 연출 |
| 생성 방식 | 서버 자동 생성 (비동기) | 사용자가 직접 커스터마이징 후 생성 |
| 진입점 | 갤러리 탭, 홈, 런 상세 | Route Art 상세 화면 > "애니메이션 만들기" |

### 핵심 차별점

기존 Route Art는 런 저장 시 자동으로 생성되는 **수동적 산출물**이다. Animated Route Art는 사용자가 배경과 캐릭터를 직접 선택하여 만드는 **능동적 창작 경험**을 제공한다. SNS 공유 시 정적 이미지 대비 시선을 끄는 효과가 크며, "내 러닝을 하나의 작품으로" 라는 앱의 핵심 가치를 강화한다.

## 2. 화면별 기능 명세

### 2-A. 애니메이션 제작 화면 (route-art/[id]/animate.tsx)

#### 목적
사용자가 배경 스타일과 캐릭터를 선택하고, 경로 애니메이션을 커스터마이징할 수 있도록 한다.

#### 사용자 스토리
- As a 러너, I want to 내 GPS 경로 위에서 캐릭터가 달리는 애니메이션을 만들고 싶다, so that 러닝 기록을 생동감 있게 표현할 수 있다.
- As a 러너, I want to AI로 배경 이미지를 생성하고 싶다, so that 내 러닝 경험에 맞는 분위기를 연출할 수 있다.
- As a 러너, I want to 다양한 캐릭터 중 하나를 선택하고 싶다, so that 나만의 개성을 표현할 수 있다.

#### 기능 요구사항
- FR-1: 상단에 실시간 미리보기 영역 (SVG 경로 + 배경 + 캐릭터 합성)
- FR-2: 배경 선택 섹션
  - FR-2a: 프리셋 배경 목록 (도시 야경, 공원, 해변, 산길, 우주 등 6종) -- 가로 스크롤
  - FR-2b: "AI 생성" 옵션 -- 텍스트 프롬프트 입력 필드 (최대 100자)
  - FR-2c: 프리셋 선택 시 즉시 미리보기 반영, AI 생성은 "생성하기" 후 반영
- FR-3: 캐릭터 선택 섹션
  - FR-3a: 프리셋 스프라이트 목록 (기본 러너, 닌자, 로봇, 고양이, 유니콘 등 6종) -- 가로 스크롤
  - FR-3b: 각 캐릭터에 달리기 애니메이션 프레임 (idle, run 2~4프레임)
  - FR-3c: 캐릭터 선택 시 미리보기에 즉시 반영
- FR-4: 애니메이션 속도 조절 슬라이더 (0.5x ~ 3x, 기본 1x)
- FR-5: "생성하기" 버튼 -- 모든 선택 완료 후 활성화

#### 비기능 요구사항
- 미리보기 렌더링: react-native-reanimated + react-native-svg 기반, 60fps 목표
- AI 배경 생성 응답 시간: 10초 이내 (FLUX.1-schnell 기준)
- 프리셋 배경/캐릭터 에셋은 앱 번들에 포함 (오프라인 사용 가능)
- AI 배경 생성은 네트워크 필수

#### 범위 외 (Out of Scope)
- 사용자 커스텀 캐릭터 업로드
- 배경 이미지 직접 그리기/편집
- 경로 색상/두께 커스터마이징 (P2로 검토)
- 음악/사운드 추가

#### 와이어프레임

```
+------------------------------------------+
|  [<뒤로]     애니메이션 만들기             |
+------------------------------------------+
|                                          |
|  +------------------------------------+  |
|  |                                    |  |
|  |     [실시간 미리보기 영역]           |  |
|  |     배경 + SVG 경로 + 캐릭터        |  |
|  |     300 x 300                      |  |
|  |                                    |  |
|  +------------------------------------+  |
|                                          |
|  배경 스타일                              |
|  +------+ +------+ +------+ +------+    |
|  |도시  | |공원  | |해변  | |AI    |    |
|  |야경  | |      | |      | |생성  | >> |
|  +------+ +------+ +------+ +------+    |
|                                          |
|  [AI 프롬프트: "벚꽃이 흩날리는 봄 공원"]   |
|                                          |
|  캐릭터 선택                              |
|  +------+ +------+ +------+ +------+    |
|  |[러너]| |닌자  | |로봇  | |고양이| >> |
|  +------+ +------+ +------+ +------+    |
|                                          |
|  속도  [----O-----------]  1.0x          |
|                                          |
|  [ ////////  생성하기  //////// ]         |
+------------------------------------------+
```

### 2-B. 애니메이션 생성 중 화면

#### 목적
AI 배경 생성 + GIF 렌더링 과정에서 사용자에게 진행 상태를 보여준다.

#### 사용자 스토리
- As a 러너, I want to 생성 진행 상황을 알고 싶다, so that 기다리는 동안 불안하지 않다.

#### 기능 요구사항
- FR-1: 단계별 진행 표시
  - (1) "배경 이미지 생성 중..." (AI 생성 선택 시만)
  - (2) "애니메이션 렌더링 중..."
  - (3) "GIF 생성 중..."
- FR-2: 각 단계에 예상 소요 시간 표시 (배경 ~10초, 렌더링 ~5초, GIF ~5초)
- FR-3: 중간에 취소 가능 (생성 중 "취소" 버튼)
- FR-4: 실패 시 에러 메시지 + "다시 시도" 버튼

#### 와이어프레임

```
+------------------------------------------+
|                                          |
|                                          |
|          [캐릭터 스프라이트 애니메이션]     |
|           (달리는 모습 반복)               |
|                                          |
|      배경 이미지 생성 중...               |
|      [=========>         ]  60%          |
|                                          |
|      예상 소요: 약 15초                   |
|                                          |
|           [취소]                          |
|                                          |
+------------------------------------------+
```

### 2-C. 결과물 미리보기/저장 화면

#### 목적
생성된 애니메이션을 미리보기하고, 저장/공유할 수 있도록 한다.

#### 사용자 스토리
- As a 러너, I want to 완성된 애니메이션을 미리보기하고 싶다, so that 만족스러운지 확인할 수 있다.
- As a 러너, I want to 애니메이션 GIF를 카메라롤에 저장하고 싶다, so that 나중에 다시 볼 수 있다.
- As a 러너, I want to 애니메이션을 SNS에 공유하고 싶다, so that 친구들에게 자랑할 수 있다.
- As a 러너, I want to 마음에 들지 않으면 다시 만들고 싶다, so that 더 나은 결과물을 얻을 수 있다.

#### 기능 요구사항
- FR-1: GIF 애니메이션 자동 재생 (루프)
- FR-2: 하단 런 요약 정보 (거리, 시간, 페이스, 날짜)
- FR-3: "저장" 버튼 -- GIF를 카메라롤에 저장 (expo-media-library)
- FR-4: "공유" 버튼 -- Share.share로 GIF 파일 공유 (Instagram, 카카오톡 등)
- FR-5: "다시 만들기" 버튼 -- 애니메이션 제작 화면으로 복귀 (이전 설정 유지)
- FR-6: 생성된 GIF URL은 Run 모델의 `animatedRouteArtUrl` 필드에 저장

#### 비기능 요구사항
- GIF 파일 크기: 5MB 이하 목표 (해상도 512x512, 15fps, 5~10초)
- 저장 시 MediaLibrary 권한 요청 (iOS/Android 분기)

#### 와이어프레임

```
+------------------------------------------+
|  [<뒤로]       애니메이션 결과    [다시만들기] |
+------------------------------------------+
|                                          |
|  +------------------------------------+  |
|  |                                    |  |
|  |     [GIF 애니메이션 자동 재생]       |  |
|  |     캐릭터가 경로를 따라 달림        |  |
|  |     512 x 512                      |  |
|  |                                    |  |
|  +------------------------------------+  |
|                                          |
+------------------------------------------+
|  3월 28일 아침 달리기                      |
|  7.1 km  |  35:00  |  4:55/km            |
+------------------------------------------+
|                                          |
|  [    저장    ]    [    공유    ]          |
|                                          |
+------------------------------------------+
```

## 3. 기술 아키텍처

### 전체 흐름

```
[모바일 앱]                    [Fastify API]              [외부 서비스]
    |                              |                          |
    |  1. 배경/캐릭터 선택          |                          |
    |  2. POST /runs/:id/animate   |                          |
    |  --------------------------> |                          |
    |                              |  3. (AI 배경 선택 시)      |
    |                              |  POST Hugging Face API   |
    |                              |  ------------------------>|
    |                              |  <-- 배경 이미지 (PNG) ---|
    |                              |                          |
    |                              |  4. 프레임 합성 + GIF 생성  |
    |                              |  (Sharp + gifenc)         |
    |                              |                          |
    |                              |  5. GIF 업로드 (S3/로컬)   |
    |                              |                          |
    |  <-- { animatedRouteArtUrl } |                          |
    |                              |                          |
    |  6. GIF 다운로드 + 표시       |                          |
    |  7. 저장/공유                 |                          |
```

### 모바일 측 (apps/mobile)

| 요소 | 기술 | 용도 |
|------|------|------|
| 미리보기 애니메이션 | react-native-reanimated + react-native-svg | 경로 위 캐릭터 이동 실시간 렌더링 |
| 캐릭터 스프라이트 | 정적 PNG 에셋 (앱 번들) | 프리셋 6종, 각 2~4 프레임 |
| 프리셋 배경 | 정적 PNG 에셋 (앱 번들) | 프리셋 6종, 512x512 |
| GIF 표시 | expo-image 또는 react-native-fast-image | 결과 GIF 재생 |
| 저장 | expo-media-library | 카메라롤 저장 |
| 공유 | expo-sharing / Share API | SNS 공유 |

### 서버 측 (services/api)

| 요소 | 기술 | 용도 |
|------|------|------|
| AI 배경 생성 | FLUX.1-schnell (Hugging Face Inference API, 무료) | 텍스트 프롬프트 -> 512x512 PNG |
| GPS -> SVG Path | 기존 route-art 모듈 | 좌표 -> SVG path 변환 (이미 구현됨) |
| 프레임 합성 | Sharp (Node.js) | 배경 + SVG 경로 + 캐릭터를 프레임별 합성 |
| GIF 인코딩 | gifenc (경량 라이브러리) | 합성된 프레임 -> 애니메이션 GIF |
| 비동기 처리 | BullMQ (기존 큐 인프라 활용) | animate-route-art 큐 |
| 파일 저장 | 로컬 스토리지 (NAS) 또는 S3 | GIF 파일 호스팅 |

### GIF 생성 파이프라인 상세

```
POST /runs/:id/animate 요청 수신
    |
    v
[1] 입력 검증 (runId, backgroundType, characterId, speed)
    |
    v
[2] BullMQ animate-route-art 큐에 Job 등록
    |
    v
[3] Worker 처리 시작
    |
    +--> [3a] 배경 이미지 준비
    |         - 프리셋: 서버 로컬 에셋에서 로드
    |         - AI 생성: Hugging Face API 호출 (FLUX.1-schnell)
    |           프롬프트: "{userPrompt}, digital art, landscape, no text"
    |
    +--> [3b] GPS 좌표 -> SVG Path (기존 모듈)
    |
    +--> [3c] 프레임 생성 (Sharp)
    |         - 총 프레임 수: fps(15) x duration(5~10초) = 75~150프레임
    |         - 각 프레임: 배경 + SVG 경로 + 캐릭터 위치(경로상 진행률)
    |         - 캐릭터 스프라이트 프레임 순환 (run_01, run_02, ...)
    |
    +--> [3d] GIF 인코딩 (gifenc)
    |         - 512x512, 15fps, 256색 최적화
    |
    +--> [3e] GIF 파일 저장 + URL 생성
    |
    +--> [3f] DB 업데이트: Run.animatedRouteArtUrl = gifUrl
    |
    v
[4] 완료 (모바일은 폴링으로 URL 확인)
```

## 4. API 변경 사항

### 새 엔드포인트

#### POST /runs/:id/animate

애니메이션 GIF 생성을 요청한다. BullMQ 큐에 Job을 등록하고 즉시 202 응답을 반환한다.

**인증**: Bearer JWT 필수

**Request**
```json
{
  "backgroundType": "preset",
  "backgroundPreset": "city_night",
  "characterId": "runner_default",
  "speed": 1.0
}
```

| 필드 | 타입 | 필수 | 설명 |
|------|------|------|------|
| backgroundType | `"preset"` \| `"ai"` | Y | 배경 생성 방식 |
| backgroundPreset | string | 조건부 | backgroundType이 "preset"일 때 필수. `city_night`, `park`, `beach`, `mountain`, `space`, `sunset` |
| backgroundPrompt | string | 조건부 | backgroundType이 "ai"일 때 필수. 최대 100자 |
| characterId | string | Y | `runner_default`, `ninja`, `robot`, `cat`, `unicorn`, `astronaut` |
| speed | number | N | 0.5 ~ 3.0, 기본 1.0 |

**Response** `202 Accepted`
```json
{
  "data": {
    "jobId": "uuid",
    "status": "queued",
    "estimatedSeconds": 20
  }
}
```

**에러**
- `404 RUN_NOT_FOUND` -- 해당 런이 존재하지 않거나 본인 소유가 아님
- `400 INSUFFICIENT_GPS_DATA` -- GPS datapoints가 2개 미만
- `409 ANIMATION_IN_PROGRESS` -- 이미 생성 중인 Job이 있음
- `429 RATE_LIMITED` -- AI 배경 생성 분당 제한 초과

---

#### GET /runs/:id/animate/status

애니메이션 생성 진행 상태를 조회한다. 모바일에서 폴링에 사용한다.

**인증**: Bearer JWT 필수

**Response** `200`
```json
{
  "data": {
    "jobId": "uuid",
    "status": "processing",
    "step": "generating_background",
    "progress": 40,
    "estimatedRemainingSeconds": 12
  }
}
```

| status 값 | 설명 |
|-----------|------|
| `queued` | 큐 대기 중 |
| `processing` | 처리 중 (step으로 세부 단계 확인) |
| `completed` | 완료 -- `animatedRouteArtUrl` 포함 |
| `failed` | 실패 -- `errorMessage` 포함 |

| step 값 | 설명 |
|---------|------|
| `generating_background` | AI 배경 생성 중 |
| `rendering_frames` | 프레임 합성 중 |
| `encoding_gif` | GIF 인코딩 중 |

**완료 시 응답**
```json
{
  "data": {
    "jobId": "uuid",
    "status": "completed",
    "animatedRouteArtUrl": "https://..."
  }
}
```

### 기존 엔드포인트 변경

#### GET /runs/:id 응답에 필드 추가

```json
{
  "data": {
    "id": "...",
    "routeArtUrl": "...",
    "animatedRouteArtUrl": "https://..."
  }

```

#### GET /runs 목록 응답에도 동일 필드 추가

- 갤러리 탭에서 GIF 뱃지 표시에 사용

## 5. 구현 우선순위

### P0 (이번 기능의 MVP -- 1~2 스프린트)

- [ ] `POST /runs/:id/animate` 엔드포인트 + Zod 검증 스키마
- [ ] `GET /runs/:id/animate/status` 폴링 엔드포인트
- [ ] BullMQ `animate-route-art` 워커 (프리셋 배경 + 프레임 합성 + GIF 인코딩)
- [ ] Run 모델에 `animatedRouteArtUrl` 필드 추가 (Prisma 마이그레이션)
- [ ] 애니메이션 제작 화면 (`route-art/[id]/animate.tsx`) -- 프리셋 배경 + 캐릭터 선택
- [ ] 미리보기 컴포넌트 (react-native-reanimated + react-native-svg)
- [ ] 생성 중 로딩 화면 (진행률 폴링)
- [ ] 결과물 화면 (GIF 표시 + 저장 + 공유)
- [ ] Route Art 상세 화면에 "애니메이션 만들기" 버튼 추가

### P1 (MVP 이후 1 스프린트)

- [ ] AI 배경 생성 연동 (FLUX.1-schnell, Hugging Face API)
- [ ] AI 프롬프트 입력 UI + 프롬프트 전처리 (안전 필터, 접미사 자동 추가)
- [ ] 애니메이션 속도 조절 슬라이더
- [ ] 갤러리 탭 GIF 뱃지 오버레이
- [ ] 프리셋 배경/캐릭터 추가 (6종 -> 12종)

### P2 (로드맵)

- [ ] 경로 색상/두께 커스터마이징
- [ ] 갤러리 "애니메이션 있음" 필터
- [ ] GIF 대신 MP4 비디오 지원 (파일 크기 최적화)
- [ ] 캐릭터 색상 커스터마이징 (palette swap)
- [ ] 공유 시 RunMate AI 워터마크 자동 삽입
- [ ] 애니메이션 히스토리 (같은 런에 여러 애니메이션 저장)

### 보류

- 사용자 커스텀 캐릭터 업로드 (에셋 관리 부담)
- 실시간 러닝 중 애니메이션 (배터리/성능 이슈)
- 음악/사운드 트랙 추가 (GIF 포맷 한계, MP4 전환 후 검토)

## 6. 완료 조건 (Definition of Done)

### P0 MVP 기준

- [ ] 사용자가 프리셋 배경과 캐릭터를 선택하여 애니메이션 GIF를 생성할 수 있다
- [ ] 생성된 GIF를 앱 내에서 재생할 수 있다
- [ ] GIF를 카메라롤에 저장할 수 있다
- [ ] GIF를 외부 앱으로 공유할 수 있다
- [ ] 생성 중 진행 상태가 표시된다
- [ ] GPS 데이터 부족 시 적절한 에러 메시지가 표시된다
- [ ] GIF 파일 크기가 5MB 이하이다
- [ ] API 엔드포인트 테스트 커버리지 80% 이상
- [ ] `docs/api-reference.md`, `docs/data-domain.md` 업데이트 완료
